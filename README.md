# Community Chat

A self-hosted group chat application for communities. Built with React, Tauri, and MQTT — deployable by a community administrator with a single command.

---

## What It Is

A lightweight, native desktop chat app. Members of a community install the client, connect to their community's server, and can chat in a shared group channel. Direct messaging between members comes in a later phase.

The server is a Mosquitto MQTT broker. It routes messages between clients and nothing else. No conversation history is stored server-side. No accounts, no tracking.

---

## Architecture

```
React UI (Tauri frontend)
        ↕ Tauri commands and events
Rust (Tauri backend)
        ↕ MQTT (rumqttc)
Mosquitto broker (Docker)
```

The Rust backend manages the MQTT connection. Incoming messages are emitted as Tauri events to React. Outgoing messages are sent via Tauri commands from React to Rust, which publishes them to the broker.

## Topic Structure

```
community/general          ← group chat (Phase 1)
community/dm/{a}/{b}       ← direct messages (Phase 2)
```

---

## Message Format

```json
{
  "username": "alice",
  "content": "Hello everyone",
  "timestamp": 1234567890
}
```

---

## Membership

Members join via an invite link generated by the administrator. The invite contains the server address. No account creation, no passwords in Phase 1 — just a username chosen on first launch.

---

## Deployment

The broker runs via Docker Compose. One command brings it up.

```bash
docker compose up -d
```

The desktop client is distributed as a native installer (`.deb`, `.exe`, `.dmg`) built by Tauri.

## Roadmap

### Phase 1 — Group Chat
- Mosquitto broker via Docker Compose
- Rust MQTT client (rumqttc) connecting to broker
- Publish and subscribe to `community/general`
- React message list and input
- Username set on first launch

### Phase 2 — Direct Messages
- DM topic structure (`community/dm/{a}/{b}`)
- Member list in UI
- Click a member to open a DM conversation

### Phase 3 — Polish
- Message history (local SQLite via Tauri)
- Typing indicators
- Online presence
- Admin invite link generation

### Phase 4 — Encryption
- E2EE layer added to message payloads
- Key exchange on first contact
- No changes required to broker or topic structure
