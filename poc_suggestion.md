# Proof of Concept: FastAPI + Python GTK

To reboot the project with a focus on Python and a simpler architecture, we propose the following Proof of Concept (POC).

## 1. Tech Stack
- **Server:** FastAPI (Python)
  - Provides a robust, modern framework with built-in WebSocket support.
  - Easy to document with Swagger (Autogenerated docs).
- **Client GUI:** GTK 4 via `PyGObject` (Python)
  - Native Linux look and feel.
  - No need for complex build systems like Flutter/Dart.
- **Communication:** WebSockets
  - Real-time bidirectional message passing.
- **Encryption (Future):** `cryptography` library
  - Porting the RSA/PBKDF2 logic discovered in the legacy Dart code.

## 2. UX Flow
1. **Launch:** User runs `python client_gtk.py`.
2. **Login/Connect Screen:**
   - Text fields for `Server URL` (defaults to `ws://localhost:8000/ws`).
   - Text field for `Username`.
   - "Connect" button.
3. **Chat Interface:**
   - A scrollable list of messages.
   - A text entry at the bottom for typing.
   - An "Update" indicator (Green dot for connected, Red for disconnected).
4. **Message Flow:**
   - User types and hits Enter -> Message sent via WebSocket.
   - Server receives -> Broadcasts to all connected clients.
   - Client receives -> Appends to GTK ListBox.

## 3. Recommended Steps for Junior Developer
1. **Server Migration:** Rewrite `backend/server.py` using `FastAPI`.
2. **GTK Skeleton:** Create a basic window with a "Send" button that logs to the console.
3. **Async Integration:** Connect GTK's event loop with Python's `asyncio` to handle WebSocket messages without freezing the UI.
4. **Shared Logic:** Create a `common/` folder for shared message types/models (using Pydantic).

## 4. Why this works
- **Unified Language:** Both the server and client use Python, reducing cognitive load for a junior developer.
- **Stability:** GTK is the standard for Linux desktop apps and FastAPI is the standard for modern Python APIs.
- **Low Friction:** No compilation steps or cross-language bridge issues.
